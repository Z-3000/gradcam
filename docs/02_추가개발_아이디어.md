## 3. 추천 수정안: "단 하나의 Admin 탭 추가" ⭐⭐⭐

여러 기능을 산발적으로 개발하기보다, **Streamlit에 Admin 탭 하나를 만들어서 아래 기능을 몰아넣는 것**이 시간 대비 효과가 가장 좋습니다.

### 기능 1: 운영 로그 대시보드

| 항목 | 내용 |
|------|------|
| **구현** | sqlite3로 `inference_log.db` 생성, pandas로 읽어서 `st.dataframe`과 `st.line_chart`로 시각화 |
| **의도** | "시스템이 돌고 있는지 죽었는지 모니터링할 줄 안다"는 것을 보여줌 |
| **소요 시간** | 1-2시간 |

**로그 스키마 예시**:
```sql
CREATE TABLE inference_log (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    timestamp DATETIME DEFAULT CURRENT_TIMESTAMP,
    model_name TEXT,
    prediction TEXT,
    probability REAL,
    filename TEXT,
    processing_time REAL
);
```

### 기능 2: 대량 데이터 일괄 처리

| 항목 | 내용 |
|------|------|
| **구현** | Admin 탭 안에 "배치 작업 구역" → ZIP 파일 업로드 → 루프 돌며 추론 → CSV 다운로드 버튼 |
| **의도** | "통계청 데이터처럼 많은 양의 데이터도 처리할 수 있는 로직을 짤 줄 안다"는 것을 보여줌 |
| **소요 시간** | 1-2시간 |

**배치 처리 흐름**:
```
[ZIP 업로드] → [압축 해제] → [이미지별 루프]
                                    ↓
                            [추론 + 로그 저장]
                                    ↓
                            [결과 DataFrame]
                                    ↓
                            [CSV 다운로드 버튼]
```

### 비추천 (시간 부족 시 과감히 포기)

| 기능 | 이유 |
|------|------|
| FastAPI 엔드포인트 | 운영 직무에서는 API 개발보다 '데이터 적재/관리'가 더 중요 |
| 복잡한 데이터 검증 | 기본적인 try-except 처리면 충분 |

>

> 2. **대량 파일 배치 처리 기능**을 구현하여 단건 처리가 아닌 대용량 데이터 처리 효율성을 고려했습니다.
>
> **주제가 무엇이든 데이터를 안정적으로 처리하고 운영하는 본질은 같다고 생각합니다.**"

## 5. 기존 아이디어 (참고용)

### 아이디어 1: 추론 로그 저장 및 통계 대시보드

**아키텍처**:
```
[사용자 업로드] → [추론 실행] → [결과 로깅]
                                    ↓
                              [CSV/SQLite 저장]
                                    ↓
                              [통계 대시보드]
                              - 일별 추론 건수
                              - 모델별 예측 분포
                              - 에러율 모니터링
```

### 아이디어 2: 배치 처리 기능 (다중 이미지 업로드)

- 여러 이미지 한 번에 업로드 → 일괄 추론
- 결과를 CSV로 다운로드
- 진행률 표시 (배치 처리 경험)

### 아이디어 3: 데이터 검증 기능

- 업로드 이미지 유효성 검사 (크기, 형식, DICOM 필수 태그)
- 에러 케이스 로깅 및 리포트

### 아이디어 4: API 엔드포인트 추가 (FastAPI)

- `grandcam_core.py`를 FastAPI로 감싸서 REST API 제공
- Swagger 문서 자동 생성
