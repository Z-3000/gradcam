## 현재 프로젝트 → 채용공고 매칭 분석

| 공고 키워드 | 현재 프로젝트 | Gap |
|-------------|---------------|-----|
| 나라통계 | 의료영상 분류 | 통계 데이터 처리 경험 부족 |
| 운영/유지관리 | HuggingFace 배포 | 로깅/모니터링 기능 없음 |

## 추가 개발 아이디어 (우선순위순)

### 1. 추론 로그 저장 및 통계 대시보드 (⭐ 강력 추천)

**공고 매칭**: 운영/유지관리, 데이터 품질 검증

**아키텍처**:
```
[사용자 업로드] → [추론 실행] → [결과 로깅]
                                    ↓
                              [CSV/SQLite 저장]
                                    ↓
                              [통계 대시보드]
                              - 일별 추론 건수
                              - 모델별 예측 분포
                              - 에러율 모니터링
```

**구현 내용**:
- 추론 시마다 로그 저장 (timestamp, model, prediction, probability)
- 간단한 통계 페이지 추가 (일별 추론 수, Normal/Pneumonia 비율)
- **면접 어필**: "운영 환경에서 데이터 품질 모니터링 경험"

**예상 소요 시간**: 2-3시간

---

### 2. 배치 처리 기능 (다중 이미지 업로드)

**공고 매칭**: 대용량 데이터 처리, 백필 경험

**구현 내용**:
- 여러 이미지 한 번에 업로드 → 일괄 추론
- 결과를 CSV로 다운로드
- 진행률 표시 (배치 처리 경험)

**예상 소요 시간**: 1-2시간

---

### 3. 데이터 검증 기능

**공고 매칭**: 데이터 정합성 검증

**구현 내용**:
- 업로드 이미지 유효성 검사 (크기, 형식, DICOM 필수 태그)
- 에러 케이스 로깅 및 리포트

**예상 소요 시간**: 1시간

---

### 4. API 엔드포인트 추가 (FastAPI)

**공고 매칭**: 개발자, 시스템 연동

**구현 내용**:
- `grandcam_core.py`를 FastAPI로 감싸서 REST API 제공
- Swagger 문서 자동 생성
- **면접 어필**: "모듈화된 코드로 다른 시스템에서 재사용 가능"

**예상 소요 시간**: 3-4시간
